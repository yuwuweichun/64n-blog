## 表达式和语句(Expressions and Statements)
表达式可以满足于仅仅产生一个值，然后供其后面的代码使用。但是，语句是独立存在的，如果它不影响外部环境，那么它就是无用的。语句可能会在屏幕上显示一些内容，例如 console.log ，或者改变机器的状态，从而影响其后的语句。这些改变被称为副作用 。

在某些情况下，JavaScript 允许你省略语句末尾的分号。但在其他情况下，分号必须存在，否则下一行会被视为同一语句的一部分。何时可以安全地省略分号的规则比较复杂且容易出错。因此，本书中所有需要分号的语句都会加上分号。我建议你也这样做，至少在你更深入地了解省略分号的细微差别之前是这样。

## 绑定(Bindings)
程序如何维护内部状态？它如何记住数据？我们已经了解了如何根据旧值生成新值，但这并不会改变旧值，新值必须立即使用，否则就会再次丢失。为了捕获并保存值，JavaScript 提供了一种称为绑定 （或变量）的功能。

```js
let caught = 5 * 5;
```

这就引出了第二种语句。特殊词（ 关键字 ） let 表示该语句将定义一个绑定。它后面跟着绑定的名称，如果要立即为其赋值，则后面跟着等号 = 运算符和一个表达式。

该示例创建了一个名为 caught 绑定，并使用它来获取 5 乘以 5 所产生的数字。

定义绑定后，其名称可以用作表达式。该表达式的值即为绑定当前保存的值。例如：

```js
let ten = 10;
console.log(ten * ten);
// → 100
```

当绑定指向某个值时，并不意味着它会永远绑定到该值。可以随时对现有绑定使用 = 运算符，将其与当前值断开连接，并使其指向新值：

```js
let mood = "light";
console.log(mood);
// → light
mood = "dark";
console.log(mood);
// → dark
```

你应该把绑定想象成触手而不是盒子。它们并不包含值，而是抓住值——两个绑定可以指向同一个值。程序只能访问它仍然持有引用的值。当你需要记住某个东西时，你可以长出一条触手来抓住它，或者将你已有的触手重新连接到它上面。

当你定义一个绑定却不赋予它值时，触手就无处可抓，最终会指向虚空。如果你请求一个空绑定的值，你会得到 undefined 。

一条 let 语句可以定义多个绑定。这些定义之间必须用逗号分隔：

```js
let one = 1, two = 2;
console.log(one + two);
// → 3
```

var 和 const 这两个词也可以像 let 一样用于创建绑定。

第一个是 var （即“variable”的缩写），它是 2015 年之前的 JavaScript 中声明绑定变量的方式，当时还没有 let 。我会在下一章详细讲解它与 let 区别。现在，你只需要记住它们的功能基本相同，但由于它在某些情况下行为异常，本书中我们很少使用它。

const 代表常量（constant） 。它定义了一个常量绑定，该绑定在其生命周期内始终指向同一个值。这对于仅给值命名以便后续轻松引用的绑定非常有用。

### 变量名(Binding Names)
绑定名称可以是一个或多个字母的任意序列。数字可以作为绑定名称的一部分——例如， catch22 就是一个有效的名称——但名称不能以数字开头。绑定名称可以包含美元符号（ $ ）或下划线（ _ ），但不能包含其他标点符号或特殊字符。

具有特殊含义的词，例如 let ，是关键字 ，不能用作绑定名称。此外，还有一些词被“保留供未来版本的 JavaScript 使用”，这些词也不能用作绑定名称。关键字和保留词的完整列表相当长：

```
break case catch class const continue debugger default
delete do else enum export extends false finally for
function if implements import interface in instanceof let
new package private protected public return static super
switch this throw true try typeof var void while with yield
```

不必记住这份列表。创建绑定时如果出现意外的语法错误，请检查是否尝试定义了保留字。

## 环境 (The environment)

在给定时间点存在的绑定及其值的集合称为环境 。程序启动时，环境并非为空。它始终包含语言标准中包含的绑定，并且大多数情况下，它还包含提供与外部系统交互方式的绑定。例如，在浏览器中，存在用于与当前加载的网站交互以及读取鼠标和键盘输入的函数。

## 函数
默认环境中提供的许多值都具有函数类型。函数是一段封装在值中的程序。这些值可以用于运行封装的程序。例如，在浏览器环境中，绑定 prompt 包含一个函数，该函数会显示一个提示用户输入的小对话框。它的用法如下：

```js
prompt("Enter passcode");
```

执行函数称为调用 、 调用或应用 。您可以通过在生成函数值的表达式后添加括号来调用函数。通常，您会直接使用包含该函数的绑定名称。括号内的值会传递给函数内部的程序。在示例中， prompt 函数使用我们提供的字符串作为对话框的文本。传递给函数的值称为参数 。不同的函数可能需要不同数量或不同类型的参数。

prompt 功能在现代 Web 编程中并不常用，主要是因为你无法控制生成的对话框的外观，但它在玩具程序和实验中可能很有用。

### console.log 函数
在示例中，我使用 console.log 来输出值。大多数 JavaScript 系统（包括所有现代浏览器和 Node.js）都提供了 console.log 函数，该函数会将参数输出到文本输出设备。在浏览器中，输出会显示在 JavaScript 控制台中。浏览器界面的这一部分默认是隐藏的，但大多数浏览器会在您按下 F12 键（或在 Mac 上按下 Command + Option + I）时打开它。如果此方法无效，请在菜单中查找名为“开发者工具”或类似名称的选项。

在本书页面上运行示例（或您自己的代码）时， console.log 输出将显示在示例之后，而不是显示在浏览器的 JavaScript 控制台中。

```js
let x = 30;
console.log("the value of x is", x);
// → the value of x is 30
```

虽然绑定名称不能包含句点字符，但 console.log 却包含一个。这是因为 console.log 不是一个简单的绑定，而是一个表达式，它从 console 绑定持有的值中检索 log 属性。我们将在第 4 章详细了解这意味着什么。

### 返回值
显示对话框或在屏幕上写入文本是一种副作用 。许多函数之所以有用，正是因为它们会产生副作用。函数也可以产生值，在这种情况下，它们无需副作用也能发挥作用。例如，函数 Math.max 接受任意数量的数字参数，并返回其中的最大值。

```js
console.log(Math.max(2, 4));
// → 4
```

当一个函数产生一个值时，我们称之为返回该值。在 JavaScript 中，任何产生值的操作都是表达式，这意味着函数调用可以用于更复杂的表达式中。在下面的代码中，我们调用了 Math.min （它是 Math.max 的逆运算），并将其作为加法表达式的一部分：

```js
console.log(Math.min(2, 4) + 100);
// → 102
```

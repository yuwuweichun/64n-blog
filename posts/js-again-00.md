## 关于编程
事实证明，编程很难。基本规则虽然简单明了，但基于这些规则构建的程序往往会变得非常复杂，以至于引入了自身的规则和复杂性。某种程度上，你是在构建自己的迷宫，很容易迷失其中。

You're building your own maze,in way,and you can easily get lost in it.

64N：创建一个新项目的时候往往是轻松愉快的，但随着时间的推移，维护并基于此做新功能的开发，会变得越来越困难。也许这也正是工程化和软件工程存在的必然性。

你需要付出必要的努力，不要妄自菲薄。你只需要坚持下去，休息一下，回头看看，确保你阅读并理解了示例程序和练习。学习是件辛苦的事，但你所学到的一切都将属于你，并会让你的后续学习更加轻松。

It is up to you to make the necessary effort.

计算机本身只能执行极其简单的操作。它们之所以如此有用，是因为它们能以惊人的速度完成这些操作。程序可以巧妙地将大量此类简单操作组合起来，从而完成非常复杂的任务。

程序是思想的结晶。它构建起来无需成本，轻若无物，而且在我们敲击键盘的过程中不断扩展。但随着程序的增长，其复杂性也会随之增加。编程的精髓在于构建不会让程序员感到困惑的程序。最好的程序既能实现有趣的功能，又能保持易于理解。

A program is a building of thought.

有些程序员认为，应对这种复杂性的最佳方法是在程序中只使用少量已被充分理解的技术。他们制定了严格的规则（“最佳实践”），规定了程序应该是什么样子，并小心翼翼地待在他们设定的安全小范围内。

这不仅枯燥乏味，而且效率低下。新问题往往需要新的解决方案。编程领域尚处于起步阶段，发展迅猛，其多样性足以容纳各种截然不同的方法。程序设计中存在许多可怕的错误，你应该至少犯一次，以便真正理解它们。对优秀程序的理解是通过实践培养的，而不是从一堆规则中学习的。

64N：同样的：在最初接触某个领域的技术或者是工具时，不必焦虑的寻找最佳实践，有些错误你至少犯一次，不然你可能只会迷惑——为什么要怎么做？

好的编程语言能够帮助程序员从更高的层面描述计算机需要执行的操作。它有助于省略细节，提供便捷的构建模块（例如 while 和 console.log ），允许你定义自己的构建模块（例如 sum 和 range ），并使这些模块易于组合。

64N：高级语言更接近人类的思维方式，低级语言更接近计算机的工作方式。高级语言例如 JavaScript，Python，Java 等，低级语言例如汇编语言，机器码等。高级语言更容易学习和使用，但可能会牺牲一些性能。低级语言更难学习和使用，但可以提供更高的性能。

## 什么是 JavaScript
JavaScript 于 1995 年问世，最初是为 Netscape Navigator 浏览器向网页添加程序而设计的。此后，所有其他主流图形化网页浏览器都采用了这种语言。它使得现代 Web 应用程序成为可能

64N：老婆饼里没有老婆，JavaScript 里没有 Java。名称相似并非出于明智之举，而是出于市场营销的考量。也许老婆饼之所以叫老婆饼，也是处于市场营销的考量？

在 Netscape 之外的软件采用 JavaScript 之后，一份标准文档被制定出来，用于描述 JavaScript 语言的工作方式，以便各种声称支持 JavaScript 的软件能够确保它们提供的是同一种语言。这份文档被称为 ECMAScript 标准，以负责标准化工作的 ECMA 国际组织命名。实际上，ECMAScript 和 JavaScript 这两个术语可以互换使用——它们指的是同一种语言的两种不同名称。

JavaScript 已经经历了多个版本。ECMAScript 3 版本是 JavaScript 崛起并占据主导地位期间（大约在 2000 年至 2010 年间）被广泛支持的版本。在此期间，雄心勃勃的 4 版本也在开发中，该版本计划对语言进行一系列重大改进和扩展。然而，对一种鲜活且广泛使用的语言进行如此彻底的改变，在政治上却难以奏效，因此 4 版本的开发工作在 2008 年被搁置。2009 年，规模小得多的 5 版本发布，该版本仅进行了一些无争议的改进。2015 年，6 版本发布，这是一次重大更新，其中包含了一些原本计划在 4 版本中实现的想法。此后，每年都会有新的、小的更新。

64N：ES6新特性？其中的ES6即是指ECMAScript 6，是 ECMAScript 标准的第六版，于 2015 年发布。它引入了许多新的特性和改进，例如箭头函数、类、模块、模板字符串、解构赋值等。

Web 浏览器并非 JavaScript 的唯一应用平台。一些数据库，例如 MongoDB 和 CouchDB，就使用 JavaScript 作为其脚本和查询语言。此外，一些桌面和服务器端编程平台，尤其是 Node.js 项目（ 第 20 章的主题），也提供了在浏览器之外进行 JavaScript 编程的环境。

64N：运行时，什么是运行时？运行时，runtime，运行时环境提供了程序运行所需的资源和支持，例如内存管理、输入输出处理、错误处理等。在 JavaScript 中，浏览器和 Node.js 都提供了运行时环境，使得 JavaScript 程序能够在不同的平台上运行。

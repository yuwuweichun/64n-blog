## 数据与函数
在计算机的世界里，只有数据。你可以读取数据、修改数据、创建新数据——但任何非数据的东西都无法被提及。

64N：我简单认为，所有程序，都只是数据+函数的组合。函数是对数据进行操作的工具，而数据则是函数操作的对象。

比特是任何二值事物，通常用 0 和 1 来表示。在计算机内部，它们可以表现为高低电荷、强弱信号，或者光盘表面的光泽或暗淡区域等。任何离散信息都可以简化为 0 和 1 的序列，从而用比特来表示。

## 值
要创建一个值，你只需调用它的名称即可。这很方便。你无需收集构建值所需的材料，也无需为此付费。你只需调用一个值， 嗖的一声，它就出现了。当然，值并非凭空产生。每个值都需要存储在某个地方，如果你想同时使用大量的值，可能会耗尽计算机内存。幸运的是，只有当你需要同时使用所有值时，这才会成为问题。一旦你不再使用某个值，它就会消失，留下它的位，这些位可以回收利用，作为构建下一代值的材料。

### Number
#### JS VS C++
64N：

C++中区分整数和浮点数，而整数又分short, int, long, float, double, unsighned变体等多种类型。JavaScript中只有一种数字类型：Number。它是一个双精度64位二进制格式的IEEE 754值。

在ES6中，JavaScript引入了BigInt类型，用于表示任意精度的整数。BigInt可以表示比Number更大的整数，但它不能与Number类型混合使用，运算。

此外，JS是动态类型语言，变量可以随时被赋予不同类型的值，而C++是静态类型语言，变量在声明时必须指定类型，并且在编译时进行类型检查。这一点稍后会在 自动类型转换 详细讨论。

#### Special numbers
JavaScript 中有三个特殊值，它们被视为数字，但行为却与普通数字不同。前两个是 Infinity 和 -Infinity ，分别代表正无穷和负无穷。Infinity Infinity - 1 仍然是 Infinity ，以此类推。不过，不要过分依赖基于无穷的计算。这种计算方式在数学上并不严谨，而且很快就会得到下一个特殊值： NaN 。

NaN 代表“Not a Number”，即使它本身是一个数字类型的值。例如，当您尝试计算 0 / 0、 Infinity - Infinity ，或任何其他无法产生有意义结果的数值运算时，都会得到 NaN 的结果。

### Operators
作用于两个值的运算符称为二元运算符，而只接受一个值的运算符称为一元运算符。

三元运算符是一个特殊的运算符，它接受三个值。它的语法如下：

```javascript
condition ? valueIfTrue : valueIfFalse
```
### Boolean
JavaScript 中只有一个值不等于它本身，那就是 NaN （“非数字”）。

```js
console.log(NaN == NaN)
// → false
```
### Empty Values
有两个特殊值，分别记为 null 和 undefined ，用来表示没有有意义的值。它们本身也是值，但不携带任何信息。

undefined 和 null 之间的区别是 JavaScript 设计上的偶然现象，大多数情况下无关紧要。在确实需要关注这些值的情况下，我建议将它们视为基本可以互换使用。

### Automatic Type Conversion
在引言中，我提到过 JavaScript 几乎可以接受你提供的任何程序，即使是那些执行一些奇怪操作的程序。以下表达式很好地证明了这一点：

```js
console.log(8 * null)
// → 0
console.log("5" - 1)
// → 4
console.log("5" + 1)
// → 51
console.log("five" * 2)
// → NaN
console.log(false == 0)
// → true
```

当运算符应用于“错误”类型的值时，JavaScript 会悄悄地将该值转换为所需的类型，但转换规则往往并非你所期望的。这称为类型强制转换 。第一个表达式中的 null 会变成 0 ，第二个表达式中的 "5" 会变成 5 （从字符串转换为数字）。然而，在第三个表达式中， + 运算符会先尝试字符串连接，然后再进行数值加法，因此 1 会被转换为 "1" （从数字转换为字符串）。

当一些无法直接映射到数字的值（例如 "five" 或 undefined ）被转换为数字时，会得到 NaN 值。对 NaN 进行进一步的算术运算仍然会得到 NaN ，因此，如果您在不寻常的地方遇到 NaN，请检查是否存在意外的类型转换。

使用 == 运算符比较相同类型的值时，结果很容易预测：当两个值相同时，结果应该为真，除非是 NaN 。但当类型不同时，JavaScript 会使用一套复杂且令人困惑的规则来确定如何处理。大多数情况下，它会尝试将其中一个值转换为另一个值的类型。但是，当运算符两侧出现 null 或 undefined 时，只有当两侧都为 null 或 undefined 时，结果才会为真。

```js
console.log(null == undefined);
// → true
console.log(null == 0);
// → false
```

这种行为通常很有用。当你想要测试一个值是否具有实际值而不是 null 或 undefined ，你可以使用 == 或 != 运算符将其与 null 进行比较。

如果要测试某个值是否精确等于 false ？由于自动类型转换，像 0 == false 和 "" == false 的表达式也为真。如果不想进行任何类型转换，可以使用两个额外的运算符： === 和 !== `。前者测试一个值是否精确等于另一个值，后者测试它们是否精确不相等。因此， "" === false 结果为 false，正如预期。

我建议谨慎使用三字符比较运算符，以防止意外的类型转换导致错误。但如果您确定两侧的类型相同，则使用较短的运算符也无妨。
